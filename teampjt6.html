13주차 기말 프로젝트 마감

최종 폴리싱(최적화) 및 VRChat 퍼블리싱

마지막 스퍼트: "아무리 예뻐도 렉이 걸리면 실패한 월드입니다."

조별 씬 조립과 기본 인터랙션 구현이 끝났다면, 이제는 성능(프레임 방어)과 디테일을 잡을 차례입니다.
오늘은 충돌체 및 오디오 최적화(Optimization), 무거운 조명을 구워내는 라이트맵 베이킹(Light Baking), 화면의 감성을 완성하는 포스트 프로세싱(Post Processing), 그리고 대망의 SDK Build & Publish 과정을 진행합니다.

친구들을 초대했는데 화면이 뚝뚝 끊긴다면 아무리 멋진 맵이라도 즐길 수 없습니다. 월드 업로드 전 아래 3가지는 반드시 점검해야 합니다.

1. 충돌체(Collider) 단순화 - Mesh Collider 남용 금지

유저가 벽을 통과하지 못하게 하거나, 물건을 잡기 위해 넣는 Collider. 모델링 형태를 그대로 따라가는 Mesh Collider는 연산량이 어마어마하게 높습니다. 복잡한 소품(의자, 조각상 등)은 반드시 Box Collider나 Sphere Collider 여러 개로 대충 감싸야 합니다.

  • 바닥이나 평평한 벽: Box Collider 사용
  • 기둥이나 나무통: Capsule Collider 사용
  • 복잡한 가구: 빈 GameObject를 만들고 Box Collider를 여러 개 이어 붙여서 투명한 장벽 만들기
Mesh Collider (무거움)
Box Collider (가벼움)

3. 거울(Mirror)은 기본 Off

거울은 씬을 두 번 그리기 때문에 렉의 1순위 원인입니다. 거울은 항상 기본 비활성화(SetActive False) 상태로 두고, 로컬(Local) 버튼으로 유저가 원할 때만 켜게 만드세요.

라이트 베이킹이란?

VR 환경에서 수십 개의 조명(가로등, 실내등)이 실시간으로 실시간 그림자를 계산하면 컴퓨터가 터집니다. '베이킹(Baking)'은 조명과 그림자, 빛 반사를 미리 계산하여 가구와 벽의 텍스처(표면 이미지)에 아예 그림을 그려서(구워서) 합쳐버리는 기술입니다. 연산량이 "0"에 가까워지면서 퀄리티는 폭발적으로 상승합니다.

베이킹을 시작하기 전, 오브젝트가 너무 많다면 라이트맵 해상도(Lightmap Resolution)를 10~20 정도로 낮춰서 먼저 테스트 굽기를 진행하세요. 굽는 도중에는 컴퓨터가 심하게 버벅일 수 있습니다.

월드의 조명(Baking)과 재질(Shader) 세팅이 끝났다면, 마지막으로 카메라 렌즈에 필터를 끼워 화면 전체의 색감과 시각 효과를 보정하는 포스트 프로세싱(Post Processing)을 적용합니다. VRChat 월드의 '감성'을 결정짓는 가장 핵심적인 단계입니다.

블룸 (Bloom)

밝은 빛이 렌즈 주변으로 뽀샤시하게 퍼지는 빛 번짐 효과입니다. 야경의 네온사인, 사이버펑크 월드, 마법 발광체(Emission)를 표현할 때 절대 빠질 수 없는 필수 필터입니다.

컬러 그레이딩 (Color Grading)

화면 전체의 대비(Contrast), 채도(Saturation), 색온도를 조절합니다. Tonemapping 모드를 ACES로 설정하면 실사 영화처럼 고급스럽고 묵직한 색감을 즉시 얻을 수 있습니다.

VRChat 전용 포스트 프로세싱 세팅법
  1. 하이어라키(Hierarchy) 빈 공간 우클릭 > 3D Object > Post-process Volume 생성
  2. 생성된 Volume 오브젝트의 인스펙터에서 Is Global을 체크(켜기) 합니다. (맵 전체 적용)
  3. Profile 항목 우측의 버튼을 눌러 새 프로필을 생성합니다.
  4. Add effect... 버튼을 눌러 Unity > Bloom이나 Color Grading 등을 추가하고 수치를 입맛에 맞게 조절합니다.
  5. 핵심 주의사항: Layer 설정 VRChat에서는 해당 Volume 오브젝트의 Inspector 상단 Layer가 반드시 PostProcessing으로 설정되어 있어야만 인게임에서 플레이어 카메라에 효과가 보입니다!
